Git基本概念

Git命令几个专用名词的译名如下：

  • Workspace：工作区
  • Index / Stage：暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库

十一、Git工作流最佳实践

  1. 主分支（master/main）：保持稳定，只用于发布版本
  2. 开发分支（develop）：日常开发的主要分支
  3. 功能分支（feature/*）：开发新功能
  4. 修复分支（hotfix/*）：修复生产环境的紧急问题
  5. 发布分支（release/*）：准备发布的分支

十二、常见问题解决

  1. 冲突解决：当合并分支时出现冲突，需要手动编辑冲突文件，然后提交
  2. 撤销错误提交：使用 git reset 或 git revert
  3. 忘记添加文件：使用 git commit --amend 追加文件
  4. 误删除分支：使用 git reflog 找回丢失的提交
  5. 忽略文件：在 .gitignore 文件中添加需要忽略的文件或目录
